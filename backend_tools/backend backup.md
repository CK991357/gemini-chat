这是一个非常系统化的问题。你现在已经迈出了最关键的一步（拥有了母本），接下来的**运维（Maintenance）**和**迁移（Migration）**才是长期稳定运行的核心。

Rescuezilla 的特点是**“全盘镜像”**，它不像苹果的时光机器那样可以“增量备份”（只备份修改过的文件）。**每次使用 Rescuezilla，都是一次完整的、覆盖式的备份。**

针对你的需求，我为你整理了一份**《Ubuntu 服务器全生命周期维护与迁移手册》**。请保存好这份文档。

### 第一部分：未来的备份策略（日常维护）

由于 Rescuezilla 需要重启服务器且每次都是全量备份，你不必每天都做。建议采用**“大版本冷备 + 小数据热备”**的组合策略。

#### 1. 什么时候需要重新做 Rescuezilla 备份？
建议在以下时间点，插入 U 盘和移动硬盘，按照之前的步骤重新做一次完整备份：
*   **重大系统更新前：** 比如从 Ubuntu 22.04 升级到 24.04。
*   **重大架构变更前：** 比如你要重新部署一套复杂的 Docker 环境，或者修改网络底层设置。
*   **固定周期：** 建议 **每 3-6 个月** 做一次。
*   **操作方式：** 和今天做的一模一样。为了节省空间，你可以删除掉移动硬盘里最早期的那个备份文件夹，只保留最近的 1-2 个版本。

#### 2. 日常“小更新”怎么办？（无需关机）
如果你只是修改了几个配置文件，或者 Docker 里产生了一些新数据，没必要重启做全盘备份。
*   **手动备份核心数据：**
    定期把你 Docker 的挂载目录（Volumes）或者代码目录，直接通过 SFTP/SCP 复制到你的电脑或 NAS 上存着即可。
*   **为什么要这样做？**
    如果服务器挂了，你用 Rescuezilla 恢复到上个月的状态，然后把这些手动备份的“最新数据”覆盖进去，系统就变成了“今天”的状态。

### 第二部分：迁移到新电脑（完整复活教程）

这是你最关心的部分。假设你买了一台全新的 NUC 或台式机，里面可能还是 Windows，如何用备份覆盖它？

#### 步骤 1：物理连接与环境准备
1.  **准备新电脑：** 确保新电脑的硬盘容量 **大于或等于** 旧电脑的硬盘容量（500GB）。
2.  **插入设备：** 插上 **Rescuezilla 启动 U 盘** 和 **移动硬盘**。
3.  **启动：** 开机狂按 F12（或新电脑对应的启动键），进入 Rescuezilla 系统。

#### 步骤 2：执行恢复 (Restore)
1.  点击桌面的黄色 **“还原 (Restore)”** 图标。
2.  **选择镜像源：** 选中你的移动硬盘，找到你最新的那个备份文件夹。
3.  **选择目标盘：** 选中新电脑的内置硬盘。
4.  **确认覆盖：** 系统会提示“目标盘数据将被抹除”，点击确认。
5.  等待进度条走完（Restore Completed）。

#### 步骤 3：调整分区大小（如果新硬盘更大）
*如果你的新硬盘是 1TB，而旧备份只有 500GB，恢复后会有 500GB 闲置。*
1.  **不要急着重启！** 在 Rescuezilla 桌面，双击打开 **GParted**（分区工具）。
2.  找到你的系统分区（通常是最大的那个 ext4 分区）。
3.  右键点击它，选择 **Resize/Move**。
4.  拖动右边的滑块，把它拉满整个条形图（占满剩余空间）。
5.  点击 **Resize**，然后点击工具栏上的 **Apply (打钩)**。
6.  操作完成后，关闭 GParted，关机，拔设备。

### 第三部分：迁移后的“水土不服”修复（网络与 Cloudflare）

这是迁移后唯一需要手动干预的地方。因为新电脑的网卡硬件变了，Ubuntu 可能会“不认识”新网卡，导致没网。**只要解决了网络，Cloudflare Tunnel 就会自动恢复。**

#### 1. 为什么 Cloudflare 会断？
你的 Cloudflare Tunnel 配置文件（token、cert）都在备份里，**它们是完好无损的**。
但是，Tunnel 需要联网才能去连接 Cloudflare 的服务器。如果新电脑连不上网，Tunnel 自然就报错。

#### 2. 修复网络配置（Netplan）实操
启动新电脑，登录进入终端（此时可能没网，Ping 不通百度）。

**A. 查看新网卡名字**
输入命令：
```bash
ip addr
```
*   你可能会看到 `lo` (本地回环)，以及另一个名字，比如 `enp3s0`、`eth0` 或 `eno1`。
*   *记住这个新名字！* （假设旧电脑叫 `enp0s25`，新电脑叫 `eno1`）。

**B. 修改配置文件**
Ubuntu 使用 Netplan 管理网络。输入以下命令编辑配置：
```bash
cd /etc/netplan/
ls  # 查看文件名，通常是 00-installer-config.yaml 或类似名字
sudo nano 00-installer-config.yaml # 编辑它
```

**C. 修改内容**
你会看到类似这样的内容：
```yaml
network:
  ethernets:
    enp0s25:  <-- 这里是旧网卡名字，新电脑不认识它
      dhcp4: true
  version: 2
```
把 `enp0s25` 改成你刚才查到的新名字（比如 `eno1`）：
```yaml
network:
  ethernets:
    eno1:     <-- 改成新的
      dhcp4: true
  version: 2
```
*   按 `Ctrl + O` 保存，回车，`Ctrl + X` 退出。

**D. 应用更改**
输入：
```bash
sudo netplan apply
```
如果没报错，试着 `ping baidu.com`。如果通了，恭喜你，修复完成！

#### 3. 验证 Cloudflare Tunnel
一旦网络通了：
*   如果你的 Tunnel 是用 **Docker** 跑的：Docker 会自动重启容器，Tunnel 会自动重连。
*   如果你的 Tunnel 是 **Systemd 服务** (`cloudflared service`)：它也会自动重试并连接。
*   **你不需要重新登录 Cloudflare，也不需要重新授权，一切设置都会自动生效。**

| 阶段 | 场景 | 操作指南 | 需要工具 |
| :--- | :--- | :--- | :--- |
| **日常** | 安装小软件、修改小配置 | 正常使用，无需备份。重要数据手拷贝一份。 | 无 |
| **维护** | 每半年 / 系统大升级前 | **重启进 Rescuezilla 做全盘备份**。可删除旧备份。 | U盘 + 移动硬盘 |
| **灾难** | 系统崩溃 / 只有报错 | 进 Rescuezilla，点 **Restore** 恢复到上个版本。 | U盘 + 移动硬盘 |
| **迁移** | 换新电脑 | 1. Rescuezilla **Restore** 到新硬盘。<br>2. 用 GParted 扩容分区。<br>3. 开机修改 Netplan 网卡名称。 | U盘 + 移动硬盘 |
